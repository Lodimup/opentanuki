Re-auth Claude

{% include "scheduler/_card_start.html" with title="Why this page?" icon="info" extra_class="mb-5" %}

OpenTanuki shells out to claude -p as same OS user that runs celery worker. CLI keep OAuth token in ~/.claude/ + macOS keychain. Token expire → run fail. Banner appear automatic. Pick option below.

{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Option 1 — Interactive login (subscription)" icon="log-in" extra_class="mb-5" %}

SSH or open Terminal on host. Run:

claude /login

Browser opens. Token cached in keychain. No re-login until expire.

{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Option 2 — Long-lived token (subscription)" icon="key" extra_class="mb-5" %} 
claude setup-token

Persistent token. Survive reboot. Best for headless.

{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Option 3 — Paste token in app (no terminal)" icon="key-round" extra_class="mb-5" %}

Run claude setup-token on any machine, copy printed token, paste into Settings. Worker inject as CLAUDE_CODE_OAUTH_TOKEN env var on each run. Bypass keychain entirely.

{% url 'settings' as settings_url %}{% include "scheduler/_btn.html" with type="link" href=settings_url label="Open Settings" icon="settings" variant="primary" extra="inline-flex" %}

Best for headless / ssh-only / docker.

{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Option 4 — Anthropic API key (env)" icon="terminal" extra_class="mb-5" %}

Or paste API key in Settings, OR export before celery worker:

export ANTHROPIC_API_KEY=sk-ant-…
uv run celery -A app worker -l info

Pay-per-token. No subscription needed.

{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Where token lives" icon="folder-key" extra_class="mb-5" %}
  • ~/.claude/ — metadata, settings.
  • macOS keychain entry Claude Code-credentials (account = your email).
  • CLI read both on every invocation. Zero config in OpenTanuki.
{% include "scheduler/_card_end.html" %} {% include "scheduler/_card_start.html" with title="Caveats" icon="alert-triangle" extra_class="mb-5" %}
  • Don't mix auth methods. If ANTHROPIC_API_KEY set after setup-token, CLI prefer env var. Confusing billing path. Pick one.
  • Don't symlink ~/.claude/. Keychain index lock to original path. Move = re-auth.
  • SSH-only sessions. If celery worker launch via ssh without GUI login, macOS keep keychain locked → CLI can't read token. Symptom: auth banner reappear after fresh boot.
  • Reboot. After macOS reboot, keychain stay locked until you log in via GUI. Worker launched on boot via launchd may fail until first GUI login.
  • Fix for headless/ssh. Either log in GUI once, OR unlock manually: 
    security unlock-keychain ~/Library/Keychains/login.keychain-db
    OR run worker via user-level launchd agent so it inherit logged-in session keychain.
  • Verify token live. 
    security find-generic-password -s "Claude Code-credentials" -w | head -c 20
claude -p "say hi"
